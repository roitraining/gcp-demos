Step 1

Log into Qwiklabs and start the Data to Insights Lab.

Step 2

Open the Google Cloud Console, and navigate to the BigQuery UI.

Step 3

In a separate browser tab, open https://github.com/roitraining/gcp-demos/blob/may22/wiki_queries.sql.

Step 4

Enter the query from the github file into the BigQuery code editor, and change the term you are searching for to whatever word you like.

Step 5

Run the query. Check the Job Information pane and note the query execution time and the bytes processed.

Step 6

Modify your query, changing the table name from 1M to 10M. Rerun the query and note the job info.

Step 7

Repeat the process with the 100M, 1B, 10B and 100B tables. For the 100B table query, enter the query time into the chat window.

  • You provisioned no servers
  • You wrote no Java or Python code
  • You processed 106B rows with regex comparisons and hundreds of thousands of aggregations in under a minute!

Step 1

Make sure you have the BigQuery console open.

Step 2

In a new browser tab, open https://github.com/roitraining/gcp-demos/blob/may22/time_travel.sql .

Step 3

Run the first query from time_travel to see the total number of rows returned in BigQuery.

Step 4

Run the second query from time_travel to see the table as it was at an earlier time.

Step 5

Run the third query from time_travel to restore a copy of the table from that point in time into a dataset in your project

  • Time travel window is configurable 2-7 days
  • Table snapshots are a new feature on top of time travel (see https://cloud.google.com/bigquery/docs/table-snapshots-intro)
